## 🎯 文档编写规范

### 1. 通用规范

#### 文件命名
- 使用大写字母和下划线：`USER_MANUAL.md`
- 英文名称优先，可添加中文副标题
- 版本化文档添加版本号：`AUDIT_REPORT_V1.1.md`

#### 文件结构
```markdown
# 文档标题 | Document Title

**元信息**（日期、版本、作者等）

---

## 章节1
内容...

---

## 章节2
内容...
```

#### 格式要求
- 使用Markdown标准语法
- 代码块必须指定语言：\`\`\`javascript
- 使用emoji增强可读性（适度）
- 中英文混排时，英文前后加空格
- 使用表格、列表提高可读性

---

### 2. README.md 规范

#### 必备章节
1. **项目标题** - 中英双语
2. **项目描述** - 简短说明（1-2句话）
3. **核心功能** - 列表形式，标注版本
4. **技术栈** - 准确列出所有关键技术
5. **安装使用** - 最终用户和开发者分开说明
6. **项目结构** - 目录树和说明
7. **配置要求** - 环境、依赖、版本
8. **版本历史** - 最近3个版本的简要说明
9. **注意事项** - 重要警告和限制
10. **开源协议** - 许可证信息

#### 更新触发条件
- 新增核心功能
- 技术栈变更
- 架构调整
- 版本发布

---

### 3. CHANGELOG.md 规范

#### 格式标准
```markdown
## [版本号] - YYYY-MM-DD

### 新增 (Added)
- ✅ 功能描述

### 优化 (Changed)
- 🔄 变更描述

### 修复 (Fixed)
- 🐛 问题描述

### 移除 (Removed)
- ❌ 移除内容

### 废弃 (Deprecated)
- ⚠️ 即将移除的功能
```

#### 更新触发条件
- **必须更新**: 每次版本发布
- **建议更新**: 重大功能提交
- **版本号规则**: 遵循语义化版本（Semantic Versioning）
  - 主版本号：不兼容的API修改
  - 次版本号：向下兼容的功能性新增
  - 修订号：向下兼容的问题修正

#### 更新触发条件
- UI/UX变更
- 新增用户可见功能
- 操作流程变更
- 用户反馈的困惑点

- **问题记录**: 记录已知问题和解决方案

#### 更新触发条件
- 架构调整
- 核心模块变更
- 构建流程变更
- 新增开发工具或依赖

### 6. APK_BUILD_GUIDE.md 规范

#### 必备章节
1. **环境要求** - 详细的环境配置
2. **完整构建步骤** - 逐步说明
3. **常见问题** - 构建错误和解决方案

#### 更新触发条件
- 构建工具版本变更
- 构建流程变更
- 新增构建步骤
- 发现新的常见问题

## ✅ 文档质量检查清单

### 代码-文档一致性检查
- [ ] 文档中提到的功能在代码中存在
- [ ] 代码中的功能在文档中有说明
- [ ] UI描述与实际界面一致
- [ ] 技术栈描述准确
- [ ] 版本号一致（package.json、CHANGELOG.md、README.md）

### 完整性检查
- [ ] 所有必备文档存在
- [ ] 每个文档包含必备章节
- [ ] 版本历史完整
- [ ] 安装/使用步骤完整

### 准确性检查
- [ ] 命令可执行
- [ ] 路径正确
- [ ] 配置参数正确
- [ ] 代码示例可运行

### 可读性检查
- [ ] 章节结构清晰
- [ ] 使用列表和表格
- [ ] 代码块有语法高亮
- [ ] 中英文排版规范

## 🔄 文档维护流程

### 开发阶段
1. **功能开发** → 记录功能点
2. **代码提交** → 在commit message中标注影响的文档
3. **功能完成** → 立即更新相关文档

### 版本发布前
1. **代码冻结** → 触发文档审计
2. **文档审计** → 使用本规范检查所有文档
3. **问题修正** → 修正所有不一致问题
4. **版本更新** → 更新CHANGELOG.md和版本号
5. **审计报告** → 生成审计报告

### 版本发布后
1. **文档归档** → 保留历史版本的审计报告
2. **用户反馈** → 根据反馈优化文档
3. **持续维护** → 发现问题立即修正

## 🚨 违规处理

### 严重违规（代码与文档严重不符）
- **定义**: 文档描述的功能不存在，或代码功能未在文档中说明
- **处理**: 立即修正，阻止版本发布

### 中等违规（格式或完整性问题）
- **定义**: 文档格式不规范，章节不完整
- **处理**: 优先修正，建议在发布前完成

### 轻微违规（优化建议）
- **定义**: 文档可读性问题，非关键信息缺失
- **处理**: 记录在案，后续优化
